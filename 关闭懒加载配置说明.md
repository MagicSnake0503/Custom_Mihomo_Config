1. 点击"延迟检测"按钮

3. 界面会显示新选中的节点

### ✅ 自动测速

## ⚠️ 如果点延迟检测后节点还是不切换

### 原因1: 配置文件未重新加载

**解决**: 重新加载配置文件

**Clash Verge:**
1. 配置 → 选中配置文件
2. 右键 → 重新加载
3. 或点击右上角刷新按钮

**ClashX Pro:**
1. 配置 → 重新加载配置文件
2. 或快捷键 Cmd+R

**Clash Meta for Android:**
1. 配置 → 长按配置
2. 更新配置

### 原因2: 延迟差距未超过容差值

**当前容差**: 50ms

**示例**:
- 当前节点: Hong Kong | 08 (39ms)
- 最快节点: Hong Kong | 02 (23ms)
- 差距: 39 - 23 = 16ms
- 判断: 16ms < 50ms → **不切换**

**解决**: 调小容差值（见下方优化建议）

### 原因3: 客户端缓存

**解决**:
1. 完全退出客户端
2. 重新启动
3. 等待自动测速完成（约10-20秒）

## 🔧 进一步优化建议

### 优化1: 调小容差值（推荐）

如果希望节点切换更灵敏：

```yaml
tolerance: 30  # 从 50 改为 30，更敏感
```

**效果**: 延迟差距超过 30ms 就会切换

### 优化2: 缩短测速间隔

如果希望更频繁地自动测速：

```yaml
interval: 180  # 从 300 改为 180（3分钟）
```

**注意**: 测速间隔太短会增加流量消耗

### 优化3: 添加快速切换间隔

有些客户端支持额外参数：

```yaml
expected-status: 204
timeout: 5000
```

## 📞 故障排除

### 问题1: 启动后不自动测速
- **原因**: 旧配置缓存

- **解决**: 检查当前使用的配置文件版本

- **原因**: tolerance 太小
- **解决**: 增大 tolerance 值（建议 30-50）
