# 📖 Mihomo 定制配置使用指南

## 📌 配置概览

这是一个根据你的需求定制的 **Mihomo/Clash Meta** 完整版配置文件，包含以下核心功能：

- ✅ **流媒体解锁**：Netflix、YouTube、Disney+、Emby 等专属策略组
- ✅ **广告拦截**：集成反广告规则，自动拦截常见广告和追踪器
- ✅ **AI 服务分流**：ChatGPT、Claude、Gemini 等 AI 服务独立策略组
- ✅ **TUN 模式**：系统级代理，接管所有流量（需管理员权限）
- ✅ **自动测速**：按地区自动选择最快节点，减少手动干预

---

## 🚀 快速开始

### 第一步：填入订阅链接

打开 `Custom_Mihomo_Config.yaml` 文件，找到以下部分：

```yaml
proxy-providers:
  Subscribe1:
    <<: *BaseProvider
    url: "http://your-service-provider-url"  # 请替换为你的订阅链接
```

将 `http://your-service-provider-url` 替换为你的机场订阅链接。

**多个订阅示例：**
```yaml
proxy-providers:
  Subscribe1:
    <<: *BaseProvider
    url: "https://example.com/sub1"
    override:
      additional-prefix: "[机场1] "
      
  Subscribe2:
    <<: *BaseProvider
    url: "https://example.com/sub2"
    override:
      additional-prefix: "[机场2] "
```

### 第二步：导入配置

#### Windows/macOS 客户端
- **Clash Verge/Clash Verge Rev**：配置 → 导入配置 → 选择文件
- **ClashX Pro**：配置 → 托管配置 → 本地文件

#### Android 客户端
- **Clash Meta for Android**：配置 → 导入配置文件
- **ClashMetaForAndroid**：点击 + 号 → 从文件导入

#### iOS 客户端
- **Stash**：首页 → 配置 → 安装配置 → 文件方式
- **Shadowrocket**：首页 → 配置文件 → 从文件导入

## ⚙️ 重要配置说明

### TUN 模式

TUN 模式**默认已启用**，提供系统级代理：

```yaml
tun:
  enable: true  # 改为 false 可关闭 TUN 模式
```

**注意事项：**
- Windows/macOS 需要管理员权限
- 启用后会接管所有系统流量
- 如遇到问题，可暂时关闭 TUN 模式

### 广告拦截

配置已集成两大广告拦截规则：
- Loyalsoldier 拒绝规则
- ACL4SSR 广告拦截规则

**测试广告拦截效果：**
- https://adblock-tester.com
- https://blockads.fivefilters.org

### 端口配置

```yaml
port: 7890           # HTTP 代理端口
socks-port: 7891     # SOCKS5 代理端口
mixed-port: 7892     # 混合端口（推荐使用）
```

建议使用混合端口 `7892`，同时支持 HTTP 和 SOCKS5 协议。

### 外部控制面板

```yaml
external-controller: 127.0.0.1:9090
```

访问地址：http://127.0.0.1:9090/ui

推荐面板：
- **Yacd**：https://yacd.haishan.me
- **Clash Dashboard**：http://clash.razord.top

## 🔧 进阶自定义

### 修改节点筛选规则

配置文件使用正则表达式筛选节点，如果机场节点命名特殊，可以修改过滤规则：

```yaml
FilterHK: &FilterHK '^(?=.*(?i)(🇭🇰|香港|港|HK|Hong))(?!.*(回国|校园|游戏)).*$'
```

### 添加自定义规则

在 `rules` 部分添加自定义规则：

```yaml
rules:
  # 自定义直连规则
  - DOMAIN-SUFFIX,example.com,DIRECT
  
  # 自定义代理规则
  - DOMAIN-KEYWORD,google,🌍 国外流量
  
  # 已有规则...
```

### 调整测速参数

```yaml
url-test:
  interval: 300      # 测速间隔（秒），默认 5 分钟
  tolerance: 50      # 容差值（毫秒），延迟差小于此值不切换
  lazy: true         # 懒加载，启动时不立即测速
```

## 🛠️ 常见问题

### 1. 节点列表为空？

**原因：** 订阅链接未填写或格式错误

**解决：**
- 检查 `proxy-providers` 部分的 `url` 是否填写正确
- 确认订阅链接可以正常访问
- 查看客户端日志，确认订阅是否下载成功

### 2. TUN 模式无法启动？

**原因：** 缺少管理员权限或系统不支持

**解决：**
- Windows：右键以管理员身份运行客户端
- macOS：在系统设置中授予完全磁盘访问权限
- 如仍无法启动，暂时关闭 TUN 模式（`enable: false`）

### 3. 部分网站无法访问？

**原因：** 分流规则冲突或 DNS 污染

**解决：**
- 检查该域名是否被错误分流
- 在 `rules` 部分添加自定义规则
- 尝试切换 DNS 服务器

### 4. 广告拦截不生效？

**原因：** 规则未更新或广告使用新域名

**解决：**
- 客户端中手动更新规则
- 等待规则自动更新（每 24 小时一次）
- 检查 `rule-providers` 中的规则源是否可访问

### 5. 自动选择切换频繁？

**原因：** 节点延迟波动较大

**解决：**
- 增大 `tolerance` 值（如改为 100）
- 将测速间隔 `interval` 调大（如改为 600）
- 使用故障转移模式代替自动选择
